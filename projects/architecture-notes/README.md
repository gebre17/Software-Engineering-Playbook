# System Architecture Notes

A repository of deep-dive architectural decisions, trade-offs, and reference diagrams for various application modules.

## Architecture Design Records (ADRs)
We use ADRs to track key technical choices and design rationales.

### ADR Template
*   **Title:** e.g., ADR-001: Choosing PostgreSQL as primary transactional DB.
*   **Status:** [Proposed | Accepted | Superseded]
*   **Context:** What is the technical scenario/problem we are trying to resolve?
*   **Decision:** What is the chosen approach, and what were the alternatives?
*   **Consequences:** What is the positive and negative impact of this decision?
